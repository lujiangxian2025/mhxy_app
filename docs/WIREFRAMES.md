# 游戏助手 Android App 线框图

## 屏幕流程图（2 Tab 架构）

```
启动 App
   ↓
首页（Dashboard）
   ├→ 点击应用卡片 → 应用详情（App Detail）
   │                    ├→ 启动任务 → 任务选择器（Bottom Sheet）
   │                    ├→ 停止任务 → 确认对话框
   │                    └→ 编辑配置 → 参数编辑器
   │
   └→ 点击"设置" Tab → 设置（Settings）
                        ├→ 服务器配置
                        ├→ 配置模板管理
                        │   ├→ 应用配置模板
                        │   └→ 任务配置模板
                        ├→ 历史记录
                        └→ 关于
```

## 设计决策记录

### 为什么是 2 Tab 而非 3 Tab？

**原设计**：首页 / 任务 / 设置（3 Tab）

**问题分析**：
- 任务 Tab 的"运行中任务"与首页卡片信息重复
- 任务 Tab 的"快速启动"可在详情页启动任务时选择
- 任务 Tab 的"历史记录"是低频功能，可放设置页

**新设计**：首页 / 设置（2 Tab）

**收益**：
- 认知负担降低：2 Tab vs 3 Tab，决策更快
- 信息架构清晰：首页 = 操作，设置 = 配置
- 代码量减少：少一个页面，少一套状态管理
- 维护成本降低：功能集中，不分散

**哲学**：能消失的 Tab 永远比能写对的 Tab 更优雅。
